# Questions

1. General / Opening Questions
   • Walk me through your resume.
   • What are you currently focusing on technically?
   • Why frontend, not backend or full-stack?
   • What kind of internship are you actually looking for?
   • What do you expect to learn from this internship?
   • How do you usually learn a new technology?

2. JavaScript (Core)
   • What are the differences between var, let, and const?
   • Explain closures in JavaScript.
   • What is hoisting?
   • What is the event loop?
   • How does JavaScript handle async code?
   • Difference between == and ===?
   • What are promises?
   • What is async/await under the hood?
   • What happens when an error is thrown inside an async function?
   • Explain reference vs value in JavaScript.
   • How does garbage collection work in JS (high level)?
   • What is a memory leak in JavaScript?
   • How do you avoid unnecessary re-renders?

3. React
   • Why React? What problem does it solve?
   • What is a component?
   • Functional components vs class components?
   • What are hooks?
   • Explain useState.
   • Explain useEffect.
   • When does useEffect run?
   • What causes re-renders in React?
   • What is prop drilling?
   • How do you share state between components?
   • What is reconciliation?
   • What is the virtual DOM?
   • How does React decide what to re-render?
   • What is a key in lists, and why does it matter?
   • What happens if you mutate state directly?
   • How do you structure a React project?
   • What mistakes have you personally made in React?

4. Next.js (This Will Be Targeted)
   • Why use Next.js instead of plain React?
   • What is server-side rendering?
   • What is static site generation?
   • Difference between SSR, SSG, and CSR?
   • What is the Next.js routing system?
   • What is the App Router vs Pages Router?
   • What runs on the server vs the client in Next.js?
   • How does hydration work?
   • What are API routes?
   • What happens during a Next.js build?
   • How does Next.js improve performance?
   • What problems did you face using Next.js?
   • Tell me about the MedFree project.
   • What was your exact contribution to MedFree?
   • What would you change if you rebuilt MedFree today?

5. Go
   • Why are you learning Go?
   • What kind of programs have you written in Go?
   • What is a pointer in Go?
   • How are pointers different in Go vs C?
   • What is stack vs heap in Go?
   • How does Go manage memory?
   • What happens when you pass a struct to a function?
   • What is nil in Go?
   • What are slices vs arrays?
   • What is a map?
   • What is a goroutine? (even if you haven’t used it deeply)
   • What is concurrency vs parallelism?
   • What did you find confusing about Go at first?
   • Where would you not use Go?

6. C / Low-Level Foundations
   • Why did you learn C?
   • What is a pointer?
   • What is pointer arithmetic?
   • Stack vs heap in C?
   • What causes segmentation faults?
   • What is undefined behavior?
   • How is memory freed in C?
   • What is the difference between malloc and calloc?
   • What happens if you forget to free memory?
   • How does C influence how you write Go or JS?
   • Why is understanding memory useful for frontend work?

7. Git & Version Control
   • What is Git?
   • Difference between Git and GitHub?
   • What is a commit?
   • What is a branch?
   • What is git remote?
   • What happens when you git pull?
   • Difference between pull and fetch?
   • What is a merge conflict?
   • How do you resolve one?
   • What is your usual Git workflow?
   • Have you ever broken a repo?
   • How do you organize commits?

8. Cloudflare / Deployment / Infra
   • What is DNS?
   • What is Cloudflare?
   • Why did you use Cloudflare?
   • What is a domain?
   • What is a subdomain?
   • How does DNS resolution work (high level)?
   • What records have you configured? (A, CNAME, etc.)
   • How long does DNS propagation take?
   • What problems did you face while setting up your domain?
   • Why not just use default GitHub Pages DNS?
   • What does Cloudflare actually sit between?

9. Terminal / Linux / Environment
   • Why terminal-centric workflow?
   • What shell do you use?
   • What commands do you use daily?
   • Difference between relative and absolute paths?
   • What is SSH?
   • How does SSH authentication work (basic idea)?
   • What is YAML used for?
   • Where have you seen YAML in real tools?
   • What is Docker used for?
   • What is Kubernetes (very high level)?
   • Why are Docker and Kubernetes important even for frontend?

10. Projects (Personal Website)
    • Why did you build your personal website?
    • What tech stack did you use?
    • How is it hosted?
    • What did you configure manually?
    • What broke during setup?
    • How do you update it?
    • What would you improve?
    • Why should I look at your website?

11. Achievements / Robotics (They Will Ask)
    • What was your role as team lead?
    • What exactly did you work on in those competitions?
    • What technical skills came from robotics?
    • How does robotics experience help in software?
    • How did you handle disagreements in the team?
    • What went wrong during competitions?
    • What did you learn from losing (runner-up)?

12. Weak / Legacy Tech (Trap Questions)
    • You listed C++ / C# / Python / SQL — how well do you know them?
    • When was the last time you used C#?
    • Why include them if you don’t use them?
    • If we ask you something in Python, how comfortable are you?
    • Are you willing to relearn them if needed?

13. Intern-Level Reality Checks
    • How do you debug when something doesn’t work?
    • What do you do when you’re stuck?
    • How do you read documentation?
    • How do you handle feedback?
    • How do you handle code reviews?
    • How do you estimate how long a task will take?
    • Have you worked with deadlines?
    • How do you balance college and work?

14. Behavioral / Personality (Quiet but Real)
    • What kind of teammate are you?
    • Do you prefer working alone or in a team?
    • How do you react when someone says your code is bad?
    • What frustrates you while coding?
    • What motivates you to keep learning?
    • What are you bad at right now?
    • Where do you think you’ll struggle in this internship?

15. Meta / Curveball
    • Why should we take you over someone else?
    • What would you do if you fail this internship?
    • What do you think you overestimate about yourself?
    • What do you think you underestimate?
    • If we gave you a task outside your comfort zone, what would you do?
    • What does “good code” mean to you?
